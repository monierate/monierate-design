Monierate
dense rate data in Crayola blue on clean white

Monierate renders currency-exchange data as a utilitarian, airy dashboard-site. White hairline-bordered cards float on near-white section canvases; a single saturated blue (#1F75FE) carries every interactive affordance while green and red are rationed strictly to rate movement. Typography is a three-instrument system with strict division of labor: Onest (600, -1% tracking) speaks in headings, buttons, and tab pills; Instrument Sans carries all body and UI text; JetBrains Mono owns every numeral — ticker prices, % changes, rate cells — so columns of naira figures align like a terminal. Components read as flat and data-first: 1px strokes, whisper shadows, 5px radii on inputs, 10px on cards, pill-shaped primary actions. Long-scroll pages always resolve into the same closing sequence — news, converter, CTA banner, deep footer.

Guidelines

Do
  • ✓Keep the font division of labor: Onest for h1–h6 and buttons (600), Instrument Sans for all body/UI (400), JetBrains Mono for every rate numeral (700 in tables)
  • ✓Reserve #1F75FE for interactive elements only: buttons, links, active tabs, focus states, chart lines
  • ✓Ration green (#03D418) and red (#E31D1C) strictly to rate movement — never decorative
  • ✓Keep the 10px atomic unit for gaps and padding; step up through 15/20/25/30 for larger rhythm
  • ✓Set -1% letter-spacing on all text 12–32px, tightening to -1.9% on 24px section headings
  • ✓Pair 1px #F2F2F2 hairline borders with the 0 4 15 #F2F2F2 shadow — cards lift by wash, not by dark shadow
  • ✓Use pill (9999px) radius on all primary and secondary buttons, 30–36px tall, 12–14px Medium labels
  • ✓End every long-scroll page with the same sequence: Latest Exchange News → Currency Converter → CTA banner → footer
  • ✓Carry the disclaimer pattern on every rate surface: "This is for informational purposes only."
Don't
  • ✗Don't ship Aeonik TRIAL or the stray Figma kit fonts (Roboto, Satoshi, Public Sans, Codec Pro) — the stack is Onest + Instrument Sans + JetBrains Mono, nothing else
  • ✗Don't set rate figures in a proportional font — comparison columns must stay mono for vertical alignment
  • ✗Don't hard-code hex values — consume the tokens; the brand blue and greys should resolve from variables
  • ✗Don't use blue on non-interactive decoration — if it's blue, it must be clickable or actively selected
  • ✗Don't introduce new grays — the ramp is fixed: #000 → #737373 → #B3B0B0 → #DDD → #ECECEC → #F2F2F2 → #F8F8F8 → #FFF
  • ✗Don't use true-black shadows on light surfaces — elevation is a gray or brand-tinted wash
  • ✗Don't drift radii (3.7 / 8.4 / 13.3 / 22.9px come from scaled components) — snap to 4/5/8/10/15/9999
  • ✗Don't set body copy at line-height 1.0 — UI text is 1.0, paragraphs are 1.5; mixing them flattens hierarchy
